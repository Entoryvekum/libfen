# libfen汉字编码输入方案优化算法
本仓库设计并实现多种优化算法，旨在为libchai等项目提供经过验证的高效方案。目前，该仓库仅用于同步和共享代码，不具备实用性。

## 模拟退火
本项目中的模拟退火使用交换字根键位、更改字根键位生成新解。固定字根被视为互斥字根，即在同一键位的固定字根不会分离，不在同一键位的固定字根不会被放置到一起。因此，固定字根可能会与最始分配相差一个排列。

## 禁忌搜索
更改字根键位后，若结果劣化，则暂时禁止重新做出这一更改。实测效果较差，难以跳出局部最优解。即使使用禁忌搜索后衔接模拟退火，解的质量也通常不如直接进行模拟退火.

## 局部随机贪心（待实现）
局部随机贪心在每一步中采样或遍历邻域。将得到的可能移动评价后排序，并使用自适应的概率分布从可选的移动中抽取并应用某一方案。这种方案是一种偏置随机化算法，是模拟退火、禁忌搜索、迭代局部搜索等算法的超集。

## 分支定界
将未被分配编码的字根视为字根本身，即可得到重码率的下界。利用这一下界即可进行分支定界对一定大小的邻域进行搜索。

## 模因算法
在遗传算法的基础上对交叉、变异后的个体进行单独优化。在本项目中，对交叉、变异后的个体实施模拟退火优化。后续也考虑使用稳态替换法。

### 贪心划分交叉算子（GPX）
GPX算子最早在用于求解图的色数的HEA算法中被提出。原始的GPX从两个父代中交替抽取最大的独立集，从而最大程度保留独立集结构。该算子结合禁忌搜索在很长一段时间里是色数求解的SOTA。重码和染色问题都是优化编码的碰撞问题，而位于同一按键的字根通常具有较少的碰撞，具有相似独立集结构。因此，本项目模仿原始GPX实现了用于输入法方案的交叉算子。交叉算子的具体流程如下：
- 在每一步中，随机选择一个父本，并选择拥有最多字根的键位
- 若子代仍有空余键位，将抽取的键位中的字根整体插入子代的空余键位；否则开始处理剩余字根
- 完成一次插入后，在两个父本中将已插入字根删除
- 对于剩余字根，可以采用随机分配或其他方法插入子代
